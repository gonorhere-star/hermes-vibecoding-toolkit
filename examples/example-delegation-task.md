# Пример: ТЗ для delegate_task (реализация)

> Универсальный шаблон. Без привязки к конкретному движку/языку.
> Заполни `[плейсхолдеры]` под свой проект.

---

```markdown
## Задача

Добавить класс **[Name]** для **[функциональность]**.

## Контекст

- **Файлы для изменения:**
  - `src/[Name].h` — заголовок
  - `src/[Name].cpp` — реализация
  - (добавь другие файлы по необходимости)
- **Конвенции проекта:** см. `principles.md` в корне
- **Зависимости:** [другие классы / модули, с которыми взаимодействует]
- **Тесты:** [путь к тестам, если нужны]

## PRE-CODE чеклист (обязательно)

> Вставь сюда секцию 1 из architecture-guardrails.

```
❌ [ЗАПРЕТ 1]
❌ [ЗАПРЕТ 2]
...
```

## Правила

1. Используй `patch` (НЕ `write_file` для существующих файлов).
2. Не нарушай запреты из PRE-CODE чеклиста.
3. Следуй конвенциям проекта (нейминг, стиль кода).
4. Верни **self-report**: что изменено, сколько строк, какие файлы.

## Критерий готовности

- Код компилируется без ошибок
- Нет нарушений PRE-CODE чеклиста
- Self-report возвращён
```

# Пример: ТЗ для delegate_task (ревью)

```markdown
## Задача

Проверить изменения в файлах:
- `src/[Name].h`
- `src/[Name].cpp`

## POST-CODE чеклист

> Вставь сюда секцию 2 из architecture-guardrails.

```
✅ Код компилируется?
✅ Нет нарушений PRE-CODE?
✅ ...
```

## Формат ответа

- ✅ **PASS** — если всё ок
- ❌ **FAIL: [файл:строка] — [что нарушено]** — если есть проблемы

## Дополнительно

Если FAIL — напиши, что именно нужно исправить в каждом конкретном месте.
```

---

## Рекомендации по формулировке ТЗ

| Элемент | Как писать |
|---|---|
| **Задача** | Одно предложение: «Добавить / Изменить / Рефакторить [что] для [зачем]» |
| **Контекст** | Пути к файлам, ссылки на конвенции, зависимости |
| **PRE-CODE** | Всегда копировать из architecture-guardrails |
| **Правила** | Инструментальные ограничения (patch vs write_file) |
| **Критерий** | Объективный: компиляция + без нарушений + self-report |

> ⚠️ **Важно:** ТЗ для реализации и ТЗ для ревью — это **разные** `delegate_task`. Не смешивай их в один вызов.
