# 🧠 Hermes VibeCoding Toolkit

**Готовый пайплайн для AI-разработки с Hermes Agent через модель «Архитектор + Исполнитель».**
Экономьте токены, сохраняйте контекст чистым, стройте проекты быстрее.

## 📋 Проблема

Типичный вайбкодинг с AI-агентом выглядит так:

1. Вы говорите «добавь систему логирования»
2. Модель читает код проекта, пишет код, компилирует, возвращает результат
3. Через 3–4 таких цикла контекст раздут до десятков тысяч токенов
4. Модель начинает забывать, что было 2 шага назад
5. Вы платите за хранение сырых данных в контексте
6. Каждая следующая итерация медленнее и дороже предыдущей

**Результат**: контекст переполнен, токены сожжены, модель ошибается в простых вещах.

## ✅ Решение

Мы разделяем разработку на две чёткие роли, которые общаются через `delegate_task`:

| Роль | Модель | Что делает | Расход токенов |
|---|---|---|---|
| **Архитектор** | GLM (дорогая) | Планирует, пишет ТЗ, анализирует отчёты | **< 5M / день** |
| **Исполнитель** | DeepSeek R1 (дешёвая) | Читает файлы, пишет код, собирает, тестирует | **> 50M / день** |

**Ключевой принцип**: GLM **никогда** не читает файлы проекта — только self-report от R1.
Это сохраняет контекст чистым, а токены — в кармане.

## 🚀 Быстрый старт

Установка занимает ~15 минут.

```bash
# 1. Клонируйте репозиторий
git clone https://github.com/your-username/hermes-vibecoding-toolkit.git
cd hermes-vibecoding-toolkit

# 2. Прочитайте концепцию
#    docs/01-concepts.md — понять подход
#    docs/02-token-discipline.md — понять правила
#    docs/05-installation.md — пошаговая инструкция

# 3. Скопируйте шаблоны навыков в профиль Hermes
cp -r templates/skills/* ~/.hermes/profiles/<name>/skills/

# 4. Настройте AGENTS.md в корне вашего проекта
cp templates/AGENTS.md /path/to/your/project/AGENTS.md

# 5. Создайте профиль Hermes (если ещё нет)
hermes profiles create gamedev
# Настройте провайдеров: GLM (архитектор) + DeepSeek R1 (исполнитель)
```

Подробнее — в [`docs/05-installation.md`](docs/05-installation.md).

## 👥 Кому подходит

| Аудитория | Почему |
|---|---|
| **Solo-разработчики** | Один AI-пайплайн заменяет половину команды |
| **Малые команды (2–3 чел.)** | Единый процесс, самодокументируемость |
| **Прототипирование** | Быстрое создание MVP без потери архитектуры |
| **Инди-разработка игр** | Автоматизация рутины, фокус на геймдизайне |
| **Open Source проекты** | Прозрачный и воспроизводимый AI-процесс |

## ⚠️ Ограничения (честно)

- **Не для простых задач** — если правка < 20 строк, overhead пайплайна больше пользы
- **delegate_task не бесплатный** — каждый запуск R1 тратит токены на загрузку контекста
- **Требует дисциплины** — один «прочитаю файл сам» и экономия исчезает
- **Не для команд > 3 человек** — нет координации между разработчиками
- **Зависит от качества R1** — галлюцинации возможны, двойное ревью снижает риск
