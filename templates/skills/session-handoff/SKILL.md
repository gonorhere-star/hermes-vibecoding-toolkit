# Session Handoff

> Shutdown-ритуал. Выполняется в конце каждой сессии.
> **ВСЕ шаги через `delegate_task` — GLM НЕ пишет файлы сам.**

---

## Триггеры запуска

Фразы пользователя, после которых запускается ритуал:
- «готово»
- «спасибо»
- «давай дальше»
- «на сегодня всё»
- «заканчиваем»

---

## Шаг 1: CHANGELOG

Обновить `CHANGELOG.md`:

```markdown
## [v0.0.NN] — YYYY-MM-DD
### Added
- [что добавили]

### Changed
- [что изменили]

### Fixed
- [что починили]

### Breaking
- [breaking changes, если есть]

### Migration
- [инструкции по миграции, если есть]
```

- Номер версии: инкремент `NN`
- Заполни секции по факту изменений

---

## Шаг 2: AGENTS.md

Обновить раздел **«Текущая сессия / следующие шаги»**:

- **Последняя:** v0.0.NN — [кратко что сделано]
- **ЮЗЕР ДОЛЖЕН:** [git commit, тестирование, проверка]
- **Следующие шаги:** [какие milestones дальше]

---

## Шаг 3: ROADMAP

Обновить статусы milestones:

- ✅ — завершено
- 🚧 — в работе
- ⬜ — не начато
- Если milestone завершён — перенести в завершённые, создать новый при необходимости

---

## Шаг 4: Git

Сформировать предложение для юзера:

> **Юзеру:** Выполни `git add . && git commit -m "v0.0.NN: [описание]"`

GLM НЕ выполняет коммит сам — только предлагает.

---

## Шаг 5: Memory (опционально)

Обновить если за сессию появились:
- Новые архитектурные конвенции
- Задокументированные грабли
- Решения, которые стоит запомнить
